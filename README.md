# 客服中转中心 - JointBERT 意图识别与槽位填充

基于 PyTorch 实现的电商客服中转/路由系统，通过意图识别和槽位填充技术，将用户需求精准引导至对应服务通道，降低大模型 token 消耗。

## 功能特点

- 🎯 **意图识别**：识别用户需求类型（退货申请、物流查询、改地址、开发票、领优惠券、转人工）
- 📦 **槽位提取**：自动提取订单号、手机号、商品名、收货地址等关键信息
- 🔄 **智能路由**：根据识别结果将用户引导至对应服务通道
- ✅ **信息确认**：支持信息补全、确认、拒绝流程
- 🌐 **Web 界面**：基于 Flask 的友好交互界面

## 使用方法

### 1. 数据预处理

```bash
python data_process.py
```

生成训练/验证/测试数据集，包含 jieba 分词、停用词过滤、BIO 标签生成。

### 2. 模型训练

```bash
python train.py
```

训练参数配置（见 config.py）：
- batch_size: 8
- learning_rate: 2e-5
- epochs: 10
- 早停策略: patience=3

### 3. 启动客服中转中心

```bash
python app.py
```

访问 http://127.0.0.1:5000 即可使用。

## 核心技术

### JointBERT 模型

- **BERT 编码器**：使用 `bert-base-chinese` 预训练模型提取语义特征
- **意图分类头**：线性层输出 6 种意图类别
- **CRF 槽位标注层**：条件随机场用于序列标注，解决标签依赖问题

### 意图识别策略

- **模型预测**：基于 BERT 的分类预测
- **规则辅助**：关键词匹配（低置信度时优先规则）
- **置信度阈值**：0.7（低于阈值时使用规则结果）

### 槽位提取策略

- **规则提取**：正则表达式匹配订单号、手机号、地址
- **模型预测**：CRF 解码 BIO 标签
- **商品词表**：常见商品名称匹配

## 项目优势

1. **降低 Token 消耗**：通过精准意图识别和槽位提取，减少大模型处理无效信息
2. **提升服务效率**：用户快速匹配到对应服务通道
3. **信息完整性检查**：自动检测缺失信息并提示用户补充
4. **双路识别机制**：模型 + 规则相结合，提升识别准确率

## 注意事项

1. 首次运行需要下载 BERT 预训练模型，约 400MB
2. 训练过程需要足够内存，建议 batch_size ≤ 8
3. 模型权重保存在 `checkpoints/best_model.pt`
4. 停用词表可根据需求扩充
